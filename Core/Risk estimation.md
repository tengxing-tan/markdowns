# ✅ User Impact & ⚙️ Technical Risk Estimation Checklist

## ✅ **User Impact Checklist**

### 1. User Scope

- [ ] How many users are affected? (e.g., all users, a subset, internal users)
- [ ] Are high-value or high-frequency users impacted?

### 2. User Experience

- [ ] Will the change improve or degrade user experience?
- [ ] Is there a learning curve or retraining required?
- [ ] Are accessibility or usability standards affected?

### 3. Business Impact

- [ ] Does this change affect revenue, conversion, or retention?
- [ ] Is it aligned with business goals or KPIs?

### 4. Communication Needs

- [ ] Do users need to be notified or educated?
- [ ] Are documentation or help resources impacted?

### 5. Fallback Plan

- [ ] Is there a way to roll back or disable the feature if needed?
- [ ] Are feature flags or staged rollouts in place?

---

## ⚙️ **Technical Risk Checklist**

### 1. Code Complexity

- [ ] Does the change touch critical or complex code paths?
- [ ] Are multiple systems or services involved?

### 2. Dependencies

- [ ] Are there external dependencies (APIs, libraries, services)?
- [ ] Are those dependencies stable and well-documented?

### 3. Testing Coverage

- [ ] Are there automated tests for this change?
- [ ] Is manual QA required?
- [ ] Are edge cases and failure modes tested?

### 4. Performance & Scalability

- [ ] Could this change impact performance or resource usage?
- [ ] Has load testing or profiling been done?

### 5. Security & Compliance

- [ ] Are there any new security risks introduced?
- [ ] Does the change comply with data protection or regulatory requirements?

### 6. Deployment Risk

- [ ] Is the deployment process well-understood and safe?
- [ ] Are rollback procedures in place?

### 7. Monitoring & Observability

- [ ] Are metrics, logs, and alerts updated for this change?
- [ ] Can issues be detected quickly post-deployment?
